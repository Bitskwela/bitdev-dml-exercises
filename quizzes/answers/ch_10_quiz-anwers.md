3. Q3 — The staging area (index) is where changes are prepared before committing.

   - Answer: **True**

4. Q4 — What is a commit?

   - Answer: **A**

5. Q5 — What is a branch in Git?

   - Answer: **A** — A parallel line of development

6. Q6 — Once a commit is pushed to a remote repository, it can never be changed.

   - Answer: **False**

7. Q7 — What is merging?

   - Answer: **C**

8. Q8 — What is a merge conflict?

   - Answer: **B**

9. Q9 — What is stashing in Git?

   - Answer: **A** — Temporarily saving changes without committing

10. Q10 — What is a remote repository?

    - Answer: **C**

11. Q11 — Pull is equivalent to running fetch followed by merge.

    - Answer: **True**

12. Q12 — What does fetch do?

    - Answer: **A** — Download changes from remote without merging

13. Q13 — You modified a file locally but haven't staged it. How do you discard the changes?

    - Answer: **C** — `git checkout -- <file>` (or equivalent)

14. Q14 — You committed changes that broke your application. What's the safest way to undo them?

    - Answer: **C** — Revert or create a new commit that undoes the bad commit

15. Q15 — What is rebasing?

    - Answer: **A** — Reapply commits on top of another base (rewrite history)

16. Q16 — What is a fast-forward merge?

    - Answer: **C**

17. Q17 — What is .gitignore used for?

    - Answer: **B** — To list files/patterns Git should ignore

18. Q18 — Git hooks allow you to run scripts automatically at certain points in Git workflow.

    - Answer: **True**

19. Q19 — What is a tag?

    - Answer: **A** — A reference to a specific commit

20. Q20 — What is a pull request?

    - Answer: **C** — A request to merge changes and request review

21. Q21 — You committed with a typo in your message. How do you fix it?

    - Answer: **B** — `git commit --amend` (if not pushed) or create a new commit

22. Q22 — Your local branch is behind the remote repository. What do you do?

    - Answer: **A** — `git pull` (fetch + merge) or rebase onto remote

23. Q23 — What is a branching strategy?

    - Answer: **C**

24. Q24 — What is cherry-picking?

    - Answer: **B** — Picking a single commit from one branch to apply to another

25. Q25 — You accidentally committed a password to a public repository. What's the best action?

    - Answer: **A** — Remove it from history and rotate the secret

26. Q26 — What is reflog?

    - Answer: **C** — A reference of local HEAD changes (history of moves)

27. Q27 — What is forking?

    - Answer: **A** — Creating your own copy of a repository to contribute

28. Q28 — Typical open-source contribution workflow?

    - Answer: **A** — Fork → branch → commit → open PR

29. Q29 — What is a protected branch?

    - Answer: **C** — A branch with restrictions (e.g., must pass CI, can't force-push)

30. Q30 — A broken feature was merged to main. How do you safely undo it?

    - Answer: **A** — Revert the merge commit

31. Q31 — You have a new project folder. What command turns it into a Git repo?

    - Answer: **git init**

32. Q32 — What command stages files for commit?

    - Answer: **git add**

33. Q33 — What command creates a commit with a message?

    - Answer: **git commit -m**

34. Q34 — What command shows branches or creates a new one?

    - Answer: **git branch**

35. Q35 — What command creates and switches to a new branch?

    - Answer: **git checkout -b**

36. Q36 — What command merges a branch into the current branch?

    - Answer: **git merge**

37. Q37 — What command pushes commits to remote?

    - Answer: **git push**

38. Q38 — What command fetches and merges remote changes?

    - Answer: **git pull**

39. Q39 — What command shows commit history?

    - Answer: **git log**

40. Q40 — What command temporarily saves uncommitted changes?
    - Answer: **git stash**
