B. snake_case
C. UPPERCASE
D. PascalCase

**Question 7:** Line continuation preferred using:

B. Parentheses grouping expression
C. Semicolon chaining
D. Double indent

**Question 8:** Docstrings are accessible via:
A. info()
B. help()
C. printdoc()
D. manual()

**Question 9:** Which improves team readability?
A. Consistent 4-space indentation
B. Random varied indentation width
C. Tabs only mixed randomly
D. Removing all blank lines

**Question 10:** Purpose of PEP 8 guidelines:
A. Enforce security controls
B. Standardize style for readability
C. Eliminate need for comments
D. Replace testing

---
## Answers
1: B  
2: B  
3: C  
4: C  
5: B  
6: B  
7: B  
8: B  
9: A  
10: B  

---
## Detailed Explanations
Q1 Indentation marks suites (blocks).  
Q2 Colon signals following block.  
Q3 Docstring stays with function, discoverable.  
Q4 Mixing leads to errors/confusion.  
Q5 Comment restates obvious code—avoid.  
Q6 snake_case standard for functions.  
Q7 Parentheses avoid fragile backslashes.  
Q8 help() introspects docstrings.  
Q9 Uniform indentation supports quick parsing.  
Q10 Shared conventions ease collaboration.
