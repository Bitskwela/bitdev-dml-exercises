# Quiz 2
## Scenario: Relationship Types
Rhea Joy designs complex schema.

**Question 6:** Foreign key purpose:
A. Unique ID
B. Links to primary key of another table
C. Optional field
D. No function

**Question 7:** Many-to-Many requires:
A. No extra table
B. Junction/association table
C. Single foreign key
D. Duplicate rows

**Question 8:** Junction table Students ↔ Courses:
A. Students(id, course_id)
B. Enrollments(student_id, course_id)
C. Courses(student_id)
D. No table needed

**Question 9:** Cardinality notation 1:N means:
A. One-to-One
B. One-to-Many
C. Many-to-Many
D. None

**Question 10:** ER diagram benefit:
A. Slower design
B. Clarifies structure before coding
C. More bugs
D. Unnecessary step

---
## Answers
1: B  
2: B  
3: B  
4: B  
5: B  
6: B  
7: B  
8: B  
9: B  
10: B  

---
## Detailed Explanations
Q1 ER diagrams visualize entities (tables) and relationships.  
Q2 Entity represents object/concept (becomes table).  
Q3 Attribute is property/column.  
Q4 Primary key uniquely identifies each row.  
Q5 One barangay has many residents (1:N).  
Q6 Foreign key references primary key of related table.  
Q7 Many-to-Many needs junction table (e.g., Enrollments).  
Q8 Enrollments(student_id, course_id) junction.  
Q9 1:N = One-to-Many.  
Q10 ER diagrams prevent design errors, clarify structure.
