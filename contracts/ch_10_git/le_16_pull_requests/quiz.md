# Quiz: Pull Requests

Answer the following questions to check your understanding of this lesson.

## Question 1

What is a pull request (PR)?

A) A command to download code
B) A request to merge your branch into another branch, with code review
C) A way to delete branches
D) A backup of your repository

**Correct Answer: B**

**Explanation:** A pull request is a collaboration feature where you propose changes and request that someone review and merge your branch into another branch (usually main).

---

## Question 2

Where do you create a pull request?

A) Using the `git pull-request` command
B) On the Git hosting platform (GitHub, GitLab, Bitbucket)
C) In your local terminal only
D) Through email

**Correct Answer: B**

**Explanation:** Pull requests are created on the web interface of hosting platforms like GitHub, GitLab, or Bitbucket, not through Git commands directly.

---

## Question 3

What should a good pull request include?

A) Just the code changes, nothing else
B) A clear title, description, and context about the changes
C) Your personal diary

**Correct Answer: B**

**Explanation:** Good PRs have descriptive titles, explain what and why you changed something, and may include screenshots or testing instructions.

---

## Question 4

True or False: Pull requests allow team members to review and discuss code before merging.

**Correct Answer: True**

**Explanation:** PRs enable code review through comments, suggestions, and discussions, improving code quality and knowledge sharing.

---

## Question 5

What happens after a pull request is approved and merged?

A) The feature branch is automatically deleted forever
B) The changes from the feature branch become part of the target branch
C) All commits are removed

**Correct Answer: B**

**Explanation:** Merging a PR integrates your changes into the target branch. The feature branch can optionally be deleted afterward.
