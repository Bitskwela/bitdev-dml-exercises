# Quiz: Branch Workflow

Answer the following questions to check your understanding of this lesson.

## Question 1

What is a Git workflow?

A) A set of rules and conventions for how a team uses branches
B) A tool that automatically writes code
C) A type of merge conflict
D) A command to speed up Git

**Correct Answer: A**

**Explanation:** A Git workflow defines how branches are created, named, merged, and managed within a team or project.

---

## Question 2

In a feature branch workflow, where should new features be developed?

A) Directly on the main branch
B) On a separate feature branch
C) In a different repository
D) Without using Git

**Correct Answer: B**

**Explanation:** Feature branches allow developers to work on features in isolation, then merge them back when complete.

---

## Question 3

What is the typical lifecycle of a feature branch?

A) Create → Delete → Merge
B) Create → Work → Merge → Delete
C) Merge → Create → Delete

**Correct Answer: B**

**Explanation:** You create a branch, do your work and commits, merge it back to main, and then delete the feature branch.

---

## Question 4

True or False: It's good practice to keep feature branches short-lived.

**Correct Answer: True**

**Explanation:** Short-lived branches reduce merge conflicts and make code reviews easier. Long-running branches tend to diverge significantly from main.

---

## Question 5

What is the purpose of branch naming conventions?

A) To make branch names longer
B) To help teams understand what each branch is for at a glance
C) To confuse new team members

**Correct Answer: B**

**Explanation:** Conventions like `feature/`, `bugfix/`, or `hotfix/` prefixes help teams quickly identify the purpose of each branch.
