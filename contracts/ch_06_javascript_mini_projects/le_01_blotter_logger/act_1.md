### Task 1: Capture Form Data

Create a function that handles form submission. It should prevent page reload, get values from the input fields, push a new entry object to the `blotterLogs` array, and clear the inputs.

```js
function handleSubmit(event) {
  event.preventDefault();

  const name = nameInput.value.trim();
  const issue = issueInput.value.trim();
  const location = locationInput.value.trim();

  blotterLogs.push({ name, issue, location });

  nameInput.value = "";
  issueInput.value = "";
  locationInput.value = "";

  renderBlotter();
}

form.addEventListener("submit", handleSubmit);
```

---

### Task 2: Render Blotter Entries

Create a function that reads the `blotterLogs` array and displays each entry inside the `#blotter-logs` container using `.map()` and template literals.

```js
function renderBlotter() {
  const htmlArray = blotterLogs.map(function (entry) {
    return `
      <div class="log-entry">
        <h4>${entry.name} @ ${entry.location}</h4>
        <p>${entry.issue}</p>
      </div>
    `;
  });

  logsContainer.innerHTML = htmlArray.join("");
}
```

### Task 3: Add Validation

Create a validation function that checks if any input is empty. If validation fails, show an alert and return `false`. Integrate this with the form handler to prevent adding empty entries.

```js
function validateInputs(name, issue, location) {
  if (!name || !issue || !location) {
    alert("Please fill in all fields!");
    return false;
  }
  return true;
}

function handleSubmit(event) {
  event.preventDefault();

  const name = nameInput.value.trim();
  const issue = issueInput.value.trim();
  const location = locationInput.value.trim();

  if (!validateInputs(name, issue, location)) {
    return;
  }

  blotterLogs.push({ name, issue, location });

  nameInput.value = "";
  issueInput.value = "";
  locationInput.value = "";

  renderBlotter();
}
```

## Breakdown of the Activity

**Variables Defined:**

- `form`: Reference to the HTML form element selected with `querySelector("#blotter-form")`.

- `nameInput`: Reference to the name input field for capturing the reporter's name.

- `issueInput`: Reference to the issue input field for capturing the complaint description.

- `locationInput`: Reference to the location input field for capturing where the incident occurred.

- `logsContainer`: Reference to the container div where blotter entries will be displayed.

- `blotterLogs`: An array that stores all blotter entry objects with `name`, `issue`, and `location` properties.

- `entry`: An object representing a single blotter log with three properties.

- `htmlArray`: An array of HTML strings generated by mapping over `blotterLogs`.

**Key Concepts:**

- `querySelector()`:
  Selects a single DOM element using CSS selector syntax. Returns the first matching element or `null`.

- `addEventListener()`:
  Attaches an event handler function to an element. The `submit` event fires when a form is submitted.

- `event.preventDefault()`:
  Prevents the default browser behavior. For forms, this stops the page from reloading on submit.

- Template Literals:
  Strings enclosed in backticks (`) that allow embedded expressions using `${expression}`. Great for creating HTML strings.

- `Array.map()`:
  Creates a new array by transforming each element. Returns an array of the same length with transformed values.

- `Array.push()`:
  Adds one or more elements to the end of an array and returns the new length.

- `Array.join()`:
  Combines array elements into a single string, with an optional separator between elements.

- `value.trim()`:
  Removes whitespace from both ends of a string. Useful for cleaning user input.

**Key Functions:**

- `handleSubmit(event)`:
  Event handler for form submission. Prevents reload, extracts input values, validates, stores data, clears inputs, and triggers re-render.

- `renderBlotter()`:
  Reads the `blotterLogs` array and updates the DOM with HTML representations of each entry using `.map()` and template literals.

- `validateInputs(name, issue, location)`:
  Checks if any required field is empty. Returns `false` and shows alert if validation fails, `true` if all fields are filled.
